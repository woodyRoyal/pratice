试用中心 v3.3 数据报表接口文档

1、获取选项配置接口：

  • 返回字段为list 则是下拉选项；string 则是input

请求URL：

  • http://e.2345.com/yuntutask/rest/api?ctl=config&type={type}

请求方式：

  • get

参数：

参数名 必选 类型 说明
{type} 是 string 配置类型 可选awake(唤醒) distribute(分发) gold(金币)

返回示例

格式如下
  {
    "httpCode":200,
    "durTimeMs":4,
    "message":"OK",
    "response":{
        "code":1,
        "msg":"success",
        "data":{
            "{type}":{//配置类型
                "{package_name}":[ //select name
                    {
                        "key":"com.xingqiu.xxx",   //option value
                        "value":"星球联盟"          //option html
                    }
                ],
                "keyword":"keyword" //input name
            }
        }
    }
}

返回参数说明

参数名 类型 说明
data object data里面包含所有的选项映射值
data.awake object 唤醒报表的筛选项
data.distribute object 分发报表的筛选项
data.gold object 金币报表的筛选

awake 类说明

参数名 类型 说明
package_name list 宿主名称
pull_target list 唤醒对象
ref_label list 位置来源
keyword string 直接填写的关键词

distribute 类说明

参数名 类型 说明
package_name list 宿主名称
task_type list 任务类型
ref_label list 位置来源
channel_id list 渠道来源
keyword string 直接填写的关键词

gold 类说明

参数名 类型 说明
package_name list 宿主名称
pull_target list 唤醒对象
ref_label list 位置来源
task_id list 任务id
channel_id list 渠道来源
task_type list 任务类型

2、获取具体数据接口：

  • PS：日期筛选统一使用date;时段筛选统一使用hour 时段支持0-23（如 0 表示 0-1 的整点） 其他筛选请使用配置接口下发的值！多选项，请使用,（英文逗号）隔开！

请求URL：

  • http://e.2345.com/yuntutask/rest/api?ctl={CTL}&{SEARCH}

请求方式：

  • get

参数：

参数名 必选 类型 说明
{CTL} 是 string 具体需要查询的报表 可选awake(唤醒) distribute(分发) gold(金币)
{SEARCH} 否 string 多个需要筛选的数据 默认不筛选
export 否 string 固定为export 添加下载请求 （注意：如果数据量少于1w，则返回所有数据，前端自行处理下载；否则返回文件下载标识符）

返回示例

格式如下
{
    "httpCode":200,
    "durTimeMs":6,
    "message":"OK",
    "response":{
        "code":1,
        "msg":"success",
        "data":{
            "search":{
                "date":"2019-12-20,2019-12-26",
                "hour":"0,2",
                "选择宿主":"xx",
                "选择唤醒对象":"xx",
                "....":"...."
            },
            "page_size":50,
            "page_num":1,
            "list":[
                {
                    "key":"value",
                    "key2":"value2"
                },
                {
                    "key":"value",
                    "key2":"value2"
                },
                {
                    "key":"value",
                    "key2":"value2"
                },
                {
                    "key":"value",
                    "key2":"value2"
                }
                ...
            ],
            "total":{
                "key":"value",
                "key2":"value2"
                ...
            },
            "ext":{
                "key":"value"
                ...
            }
        }
    }
}

返回参数说明

参数名 类型 是否必填 说明
data object 是 data里面包含所有的选项映射值
data.search object 否 请求的筛选项
data.pageSize int 是 总数据条数
data.pageNum int 是 当前页码
data.list object.list 是 当前页码的数据
data.total object 是 所有数据汇总值
data.ext object 是 概览的扩展数据

data.ext说明

参数名 类型 是否必填 说明
success_pv_today int 是 今日唤醒成功数
success_pv int 是 昨日唤醒成功数
success_ratio int 是 昨日唤醒成功率
task_gold int 是 昨日金币支出
task_uv int 是 昨日参与用户数
task_id_count int 是 昨日投放任务数
download_done_pv_today int 是 今日下载成功数
download_done_pv int 是 昨日下载成功数
download_done_pv_yyb int 是 昨日应用宝下载成功数
download_done_pv_zl int 是 昨日智量下载成功数
download_done_pv_ssp int 是 昨日自有下载成功数
task_gold int 是 昨日金币支出
task_uv int 是 昨日参与用户数
task_id_count int 是 昨日投放任务数

3、文件下载地址：

  • 文件下载

请求URL：

  • http://e.2345.com/yuntutask/rest/api?ctl=download&{SEARCH}

请求方式：

  • get

参数：

参数名 必选 类型 说明
SEARCH{} 否 string 筛选条件 日期：date=xxx,xxxx

返回示例

{
    "httpCode":200,
    "durTimeMs":6,
    "message":"OK",
    "response":{
        "code":1,
        "msg":"success",
        "data":{
            "list":[
                {
                    "date":"value",//日期
                    "filename":"value2",//文件名
                    "created_at":"value2",//创建时间
                    "status":"0",// 0=生成中  1=已完成
                    "url":"value2"//文件地址
                }
                ...
            ],
        }
    }
}
